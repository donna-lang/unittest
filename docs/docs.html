unittest

Package metadata helpers.

The assertion API lives in should.donna so users can write:

import unittest/should

This module intentionally has no main: unittest is a library package, not an application.

pub fn version() -> String

Return the package version as a string.

Keeping this tiny function gives the generated root module a useful public value without turning the package into an executable.

pub fn version_test() -> String:
  unittest.version()

should

Assertion helpers for Donna tests.

Donna's test runner expects every test function to return a String.

Convention: "" means the assertion passed. any non-empty String is printed as the failure message.

This module keeps the original unittest API intact and enriches it with more assertions for strings, bools, floats, and lists.

pub fn all(results: List(String)) -> String

Combine many assertion results into one test result.

Passing assertion results ("") are ignored. Failing results are joined with a blank line between them so each failure block remains readable.

pub fn profile_test() -> String:
  should.all([
    should.equal_int(2 + 2, 4),
    should.equal_string("Donna", "Donna")
  ])

pub fn pass() -> String

Always return a passing assertion result.

This is useful in pattern matches where one branch is explicitly the success case and the other branch returns should.fail(...).

pub fn success_branch_test() -> String:

This keeps the original unittest API exactly available while giving tests an escape hatch for custom conditions.
